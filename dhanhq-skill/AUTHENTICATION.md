## Overview

DhanHQ APIs use **JWT access tokens** passed as a header on every request. There are two user categories:

| Type | Description |
|---|---|
| **Individual** | Dhan account holders — traders, coders, algo builders |
| **Partner** | Platforms (algo platforms, fintechs, banks, PMS) building on top of DhanHQ for their users |

**Eligibility:** All Dhan users get **free access to Trading APIs**. Data APIs require an additional subscription.

---

## Access Token — Manual (Web)

The simplest method for individual traders.

1. Login to [web.dhan.co](https://web.dhan.co)
2. Go to **My Profile → Access DhanHQ APIs**
3. Click **Generate Access Token** (valid for **24 hours**)
4. Optionally enter a **Postback URL** to receive order updates

---

## Generate Token via API (TOTP required)

Programmatically generate a token if TOTP is enabled on your account.

**Endpoint:** `POST https://auth.dhan.co/app/generateAccessToken`

```bash
curl --location --request POST \
  'https://auth.dhan.co/app/generateAccessToken?dhanClientId=1000000001&pin=11111&totp=000000'
```

**Query Parameters:**

| Field | Description |
|---|---|
| `dhanClientId` | Your Dhan Client ID |
| `pin` | 6-digit Dhan PIN |
| `totp` | 6-digit TOTP code from authenticator app |

**Response:**

```json
{
    "dhanClientId": "1000000401",
    "dhanClientName": "JOHN DOE",
    "dhanClientUcc": "ABCD12345E",
    "givenPowerOfAttorney": false,
    "accessToken": "eyJ...",
    "expiryTime": "2026-01-01T00:00:00.000"
}
```

| Field | Type | Description |
|---|---|---|
| `dhanClientId` | string | User-specific ID generated by Dhan |
| `dhanClientName` | string | Name registered on Dhan |
| `dhanClientUcc` | string | Unique Client Code registered with Dhan |
| `givenPowerOfAttorney` | boolean | DDPI status |
| `accessToken` | string | JWT token for API authentication |
| `expiryTime` | string | Expiry time — 24 hours from generation |

## Renew Token

Refreshes an **active** token for another 24 hours. The old token is invalidated.
Only works on tokens generated from Dhan Web.

```bash
curl --location 'https://api.dhan.co/v2/RenewToken' \
  --header 'access-token: {JWT Token}' \
  --header 'dhanClientId: {Client ID}'
```

> **Note:** Expired tokens cannot be renewed. Only active tokens are eligible.

## API Key & Secret — OAuth Flow (Individuals)

An OAuth-based alternative for individual traders. API Key & Secret are valid for **12 months**.

**Setup:**
1. Login to web.dhan.co → My Profile → Access DhanHQ APIs
2. Toggle to **API Key** → Enter app name, Redirect URL, and optional Postback URL

### Step 1: Generate Consent

Validates the API key & secret and creates a session.

```bash
curl --location --request POST \
  'https://auth.dhan.co/app/generate-consent?client_id={dhanClientId}' \
  --header 'app_id: {API key}' \
  --header 'app_secret: {API secret}'
```

**Headers:**

| Field | Description |
|---|---|
| `app_id` *(required)* | API Key from Dhan |
| `app_secret` *(required)* | API Secret from Dhan |

**Response:**

```json
{
    "consentAppId": "940b0ca1-3ff4-4476-b46e-03a3ce7dc55d",
    "consentAppStatus": "GENERATED",
    "status": "success"
}
```

> Up to **25 consentAppId** can be generated per day. Only one token is active at a time.

---

### Step 2: Browser-Based Login

Open this URL in a browser. User logs in with Dhan credentials + 2FA (OTP/PIN/password).
On success, user is redirected to your Redirect URL with a `tokenId`.

```
https://auth.dhan.co/login/consentApp-login?consentAppId={consentAppId}
```

**Response (302 redirect):**

```
{redirect_URL}/?tokenId={Token ID for user}
```

> Extract `tokenId` from the redirect URL path parameter.

---

### Step 3: Consume Consent

Exchange the `tokenId` for an access token.

```bash
curl --location \
  'https://auth.dhan.co/app/consumeApp-consent?tokenId={Token ID}' \
  --header 'app_id: {API Key}' \
  --header 'app_secret: {API Secret}'
```

**Response:**

```json
{
    "dhanClientId": "1000000001",
    "dhanClientName": "JOHN DOE",
    "dhanClientUcc": "CEFE4265",
    "givenPowerOfAttorney": true,
    "accessToken": "<access token>",
    "expiryTime": "2025-09-23T12:37:23"
}
```

| Field | Description |
|---|---|
| `dhanClientId` | User-specific ID |
| `dhanClientName` | User's name |
| `dhanClientUcc` | Unique Client Code (UCC) |
| `givenPowerOfAttorney` | DDPI status (true/false) |
| `accessToken` | JWT token for all API calls |
| `expiryTime` | ISO timestamp of expiry (IST) |

## Partner Authentication — OAuth Flow

For partners who have received `partner_id` and `partner_secret` from Dhan.

### Step 1: Generate Consent (Partner)

```bash
curl --location 'https://auth.dhan.co/partner/generate-consent' \
  --header 'partner_id: {Partner ID}' \
  --header 'partner_secret: {Partner Secret}'
```

**Response:**

```json
{
    "consentId": "ab5aaab6-38cb-41fc-a074-c816e2f9a3e0",
    "consentStatus": "GENERATED"
}
```

---

### Step 2: User Login on Browser / WebView

```
https://auth.dhan.co/consent-login?consentId={consentId}
```

**Response (302 redirect):**
```
{redirect_URL}/?tokenId={Token ID for user}
```

---

### Step 3: Consume Consent (Partner)

```bash
curl --location \
  'https://auth.dhan.co/partner/consume-consent?tokenId={Token ID}' \
  --header 'partner_id: {Partner ID}' \
  --header 'partner_secret: {Partner Secret}'
```

**Response:** Same structure as individual OAuth Step 3 above.

## Static IP Setup

**Required by SEBI/exchange guidelines** for Order Placement APIs (Orders, Super Order, Forever Order).
Not required for fetching order/trade details.

- Set up **primary** and **secondary** IP
- Once set, cannot be modified for **7 days**
- Supports both IPv4 and IPv6

### Set IP

```bash
curl --request POST \
  --url https://api.dhan.co/v2/ip/setIP \
  --header 'Content-Type: application/json' \
  --header 'access-token: {Access Token}' \
  --data '{
    "dhanClientId": "1000000001",
    "ip": "10.200.10.10",
    "ipFlag": "PRIMARY"
  }'
```

**Body Parameters:**

| Field | Type | Description |
|---|---|---|
| `dhanClientId` *(required)* | string | Your Dhan Client ID |
| `ip` *(required)* | string | Static IP in IPv4 or IPv6 format |
| `ipFlag` *(required)* | enum | `PRIMARY` or `SECONDARY` |

**Response:**
```json
{ "message": "IP saved successfully", "status": "SUCCESS" }
```

---

### Modify IP

Same parameters as Set IP. Can only be used once every **7 days**.

```bash
curl --request PUT \
  --url https://api.dhan.co/v2/ip/modifyIP \
  --header 'Content-Type: application/json' \
  --header 'access-token: {Access Token}' \
  --data '{
    "dhanClientId": "1000000001",
    "ip": "10.200.10.10",
    "ipFlag": "PRIMARY"
  }'
```

---

### Get IP

Retrieve currently set IPs and next allowed modification date.

```bash
curl --request GET \
  --url https://api.dhan.co/v2/ip/getIP \
  --header 'access-token: {Access Token}'
```

**Response:**

```json
{
    "modifyDateSecondary": "2025-09-30",
    "secondaryIP": "10.420.43.12",
    "modifyDatePrimary": "2025-09-28",
    "primaryIP": "10.420.29.14"
}
```

| Field | Type | Description |
|---|---|---|
| `primaryIP` | string | Currently set primary static IP |
| `secondaryIP` | string | Currently set secondary static IP |
| `modifyDatePrimary` | string | Date when primary IP can be changed (YYYY-MM-DD) |
| `modifyDateSecondary` | string | Date when secondary IP can be changed (YYYY-MM-DD) |

## TOTP Setup

TOTP (Time-based One-Time Password) simplifies API-only auth. Generates a fresh 6-digit code every 30 seconds from a shared secret (RFC 6238).

**How to set up:**
1. Go to Dhan Web → DhanHQ Trading APIs → **Setup TOTP**
2. Confirm with OTP on mobile/email
3. Scan the QR code in an Authenticator app (e.g., Google Authenticator, Authy)
4. Confirm by entering the first TOTP code

Once enabled, TOTP appears as default 2FA in API key auth and partner login flows.

## User Profile (Token Validation)

Use this to verify your token is valid and check account setup. Good first test API.

```bash
curl --location 'https://api.dhan.co/v2/profile' \
  --header 'access-token: {JWT}'
```

**Response:**

```json
{
    "dhanClientId": "1100003626",
    "tokenValidity": "30/03/2025 15:37",
    "activeSegment": "Equity, Derivative, Currency, Commodity",
    "ddpi": "Active",
    "mtf": "Active",
    "dataPlan": "Active",
    "dataValidity": "2024-12-05 09:37:52.0"
}
```

| Field | Type | Description |
|---|---|---|
| `dhanClientId` | string | Your Dhan Client ID |
| `tokenValidity` | string | Token expiry date & time |
| `activeSegment` | string | Active trading segments on your account |
| `ddpi` | string | DDPI status — `Active` / `Deactive` |
| `mtf` | string | MTF consent status — `Active` / `Deactive` |
| `dataPlan` | string | Data API subscription status — `Active` / `Deactive` |
| `dataValidity` | string | Data API subscription expiry datetime |
